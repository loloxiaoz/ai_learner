# Product Spec：AI 转型工程师学习平台

**文档版本** v0.1 · 2026-03-28
**定位** 面向在校学生 + 软件工程师的 AI/大模型免费开源学习网站
**核心主张** 内容有深度、进度可视化、理解可验证

---

## 1. 目标用户

| 用户群 | 痛点 | 目标 |
|--------|------|------|
| 在校学生（CS/相关专业） | 理论多、工程感弱，不知道面试考什么 | 系统建立 AI 知识体系，准备求职 |
| 软件工程师（后端/全栈为主） | 有编码基础，缺少 AI 系统性知识，准备晋升/转岗 | 快速补齐大模型知识，能在答辩/面试中讲清楚 |

---

## 2. 核心功能模块

### 2.1 学习主线 — 10 章线性课程

- 每章对应现有 `01_正文.md` + `02_学习检验.md`
- 页面结构：**章节列表 → 正文阅读 → 学习检验（自测题）**
- 章节间有依赖关系（解锁机制可选，MVP 阶段全部开放）

### 2.2 进度看板 — 线性进度条 + 知识点 Tag 云

**进度条**
```
第 1 章 AI 基础    ████████░░  80%
第 2 章 深度学习   ██████░░░░  60%
第 3 章 Transformer ░░░░░░░░░░   0%
```
- 进度来源：用户答对自测题的比例
- 数据存 `localStorage`（无需登录，MVP 阶段）

**知识点 Tag 云**
- 预设全局知识点列表（约 60～80 个 Tag，来自 10 章内容）
- 用户答题后对应 Tag 获得「★」等级（1～5 星）
- Tag 字号 + 颜色随掌握度变化，形成热力视觉
- 示例：`[注意力机制 ★★★]` `[RLHF ★★]` `[RAG ★]`

### 2.3 交互动画 — Transformer 自注意力可视化（优先级 P0）

- 用户输入一句话（或选预设 prompt）
- 动态展示：分词 → Embedding → Q/K/V 矩阵 → 注意力分数热图 → 输出权重
- 技术方案：Canvas / D3.js + 预计算权重数据（无需实时推理）
- 后续可扩展：反向传播动画、RAG 全流程动画

### 2.4 学习检验 — 章节自测题

- 每章 10+ 题，题型：单选 + 简答（简答题提供参考答案对照）
- 答题后：每题给出详细解析
- 答题结果更新进度条 & Tag 云掌握度

## 3. 信息架构

```
/                        首页（项目介绍 + 进度看板入口）
/learn                   课程总览（10章列表 + 全局进度条 + Tag 云）

/learn/[chapter]/quiz    章节自测题
/visualize               可视化动画专区
/visualize/attention     Transformer 自注意力交互动画
```

## 4. 技术方案

| 层 | 选型 | 理由 |
|----|------|------|
| 框架 | Next.js 15 (App Router) | SSG 渲染课程内容，SEO 友好，Vercel 一键部署 |
| 样式 | Tailwind CSS v4 | 快速开发，暗色模式支持 |
| Markdown 渲染 | MDX + rehype-highlight | 代码块高亮，支持内嵌 React 组件（动画） |
| 可视化 | D3.js | 注意力热图、Tag 云、进度图 |
| 状态持久化 | localStorage（MVP） | 无需后端，用户数据本地存储 |
| 部署 | Vercel + GitHub | 开源免费，自动 CI/CD |
| 内容来源 | 直接读取本仓库 Markdown 文件 | 内容已有，无需重写 |

---

## 5. MVP 范围（第一版）

**包含：**
- [ ] 首页 + 课程总览页
- [ ] 章节正文页（Markdown 渲染，支持代码高亮）
- [ ] 章节自测题页（单选题 + 答案解析）
- [ ] 线性进度条（localStorage 持久化）
- [ ] 知识点 Tag 云（基础版，掌握度驱动）
- [ ] Transformer 注意力可视化（核心动画）

**不包含（后续迭代）：**
- 用户登录 / 云端同步进度
- 反向传播动画、RAG 动画
- AI 面试官模拟对话
- 评论 / 讨论区

---

## 6. 设计原则

1. **内容优先**：Markdown 正文是主角，动画是辅助理解的工具，不喧宾夺主
2. **无摩擦启动**：无需注册即可开始学习，进度自动保存本地
3. **工程师审美**：暗色主题为主，代码感强，避免过度卡通化
4. **开源友好**：代码 + 内容全部开源，接受 PR 贡献

## 8. 开发优先级排序

```
P0（MVP 核心）
  ├─ 课程正文渲染（读 Markdown）
  ├─ 章节自测题 + 答题逻辑
  ├─ 进度条 + Tag 云（localStorage）
  └─ Transformer 注意力动画

P1（体验提升）
  ├─ 移动端适配
  ├─ 暗色/亮色主题切换
  └─ 章节搜索

P2（内容扩展）
  ├─ 更多交互动画
  └─ 进度云端同步（需要后端）
```
